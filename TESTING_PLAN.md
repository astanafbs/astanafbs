# План тестирования приложения

## 1. Подготовка окружения

Проверить установку зависимостей и базовую сборку:

```bash
npm install
npm run infra:up
npm run db:migrate
npm run db:seed
npm run api:check
npm run admin:check
npx tsc --noEmit
```

Запустить сервисы:

```bash
npm run api:dev
npm run admin:dev
npm run start
```

Для проверки native-функций, включая WebView и ориентацию:

```bash
npx expo run:ios
npx expo run:android
```

## 2. Backend API

### Smoke

Проверить:

```bash
curl http://localhost:3000/health
curl http://localhost:3000/tournaments
curl http://localhost:3000/streams
curl http://localhost:3000/training/templates
```

### Турниры и матчи

- Список турниров загружается.
- Карточка турнира открывается.
- `/tournaments/:id/matches` возвращает матчи.
- У матчей есть `has_stream`, `stream_id`, если эфир привязан.
- Генерация сетки не ломает существующие матчи.
- Обновление результата матча работает.
- Победитель проходит дальше по сетке.

### Трансляции

Public API:

- `GET /streams` возвращает только `published` с непустым `youtube_video_id`.
- `GET /streams/:id` не отдает `draft` и `archived`.
- `GET /matches/:id/stream` возвращает эфир опубликованного матча.
- `GET /matches/:id/stream` возвращает `404`, если эфира нет.
- `GET /matches/:id/stream` возвращает `404`, если эфир в `draft`.
- `GET /matches/:id/stream` возвращает `404`, если нет Video ID.

Admin API:

- `GET /admin/streams`
- `POST /admin/streams`
- `PATCH /admin/streams/:id`
- `DELETE /admin/streams/:id`
- Один матч не должен иметь две трансляции.

### Тренировки

- `/training/templates` отдает новые названия:
  - `Разбой пирамиды`
  - `Чужой прямой удар`
  - `Свояк, скат с точки`
  - `Свояк в отскок`
  - `Чужой на резке`
- `/training/sessions` работает.
- `POST /training/sessions` сохраняет drills.
- Админка может менять training templates.

- Список матчей открывается.

- Статусы `scheduled`, `live`, `completed`, `cancelled` работают.
- Счет сохраняется.
- Победитель сохраняется.
- Дата сохраняется.

### Трансляции

В `/streams` проверить:

- `draft` не виден в приложении.
- `published` виден в приложении.
- Переключение `published`/`draft` работает.
- Редактирование Video ID работает.
- Отвязка от матча работает.
- Удаление трансляции работает.

### Тренировки

- Новые названия отображаются.
- Создание шаблона работает.
- Редактирование шаблона работает.
- `draft`-шаблон не виден в мобильном приложении.

## 4. Mobile App

### Главные разделы

Проверить на iOS и Android:

- Home
- Турниры
- Рейтинг
- Дуэли
- Профиль
- Трансляции
- Тренировочный дневник
- Новости
- Клубы

### Турнирный flow

- Список турниров.
- Карточка турнира.
- Регистрация.
- Игроки турнира.
- Сетка.
- Список матчей.
- Переход в конкретный матч.
- Отображение счета и статуса.

### Трансляции

- `/streams` показывает опубликованные эфиры.
- `/streams/:id` открывает YouTube-плеер.
- Экран матча показывает эфир, если он привязан.
- Если эфира нет, показывается нормальное сообщение.
- Если Video ID неверный, показывается ошибка.

Критично проверить WebView:

- Видео играет внутри приложения.
- Клик по YouTube/logo/watch не уводит в YouTube.
- `youtu.be`, `m.youtube.com`, `youtube.com/watch` не открываются.
- Fullscreen/landscape работает.
- При возврате со страницы ориентация снова portrait.
- Android back не ломает плеер.
- iOS swipe/back не оставляет landscape.

### Тренировки

- Новые шаблоны видны.
- Выбор шаблона меняет focus.
- Stepper длительности работает.
- Stepper оценки работает.
- Drills редактируются.
- Сохранение тренировки работает.
- История обновляется.
- Аналитика пересчитывается.

## 5. Регрессия перед релизом

Обязательные проверки:

```bash
npx tsc --noEmit
npm run api:check
npm run admin:check
npm run check
```

Если `npm run check` падает только из-за `expo-doctor`, отдельно разобрать предупреждения.

Smoke checklist:

- API поднят.
- Admin открывается.
- Mobile открывается.
- Login/onboarding не сломан.
- Турнир открывается.
- Матч открывается.
- Трансляция открывается.
- Тренировка сохраняется.
- Нет красных ошибок в Metro.
- Нет ошибок в API logs.

## 6. Сборки

Dev:

```bash
npx expo run:ios
npx expo run:android
```

Production/EAS:

```bash
npm run build:prod:ios
npm run build:prod:android
```

После сборки обязательно проверить трансляции на реальном устройстве, потому что WebView, fullscreen и ориентация могут отличаться от simulator.
